# 📈 EarningWhisperer — 실시간 어닝콜 분석 및 매매 시스템 기획

*시간과 언어의 장벽을 해소하고, 개인 투자자에게도 기관 수준의 인사이트를 제공합니다.*

### 1. 프로젝트 배경 및 필요성 (Why?)

기존 유료 터미널(Bloomberg, AlphaSense)이나 플랫폼(SeekingAlpha)이 있음에도 **EarningWhisperer**가 필요한 이유입니다.

| 구분 | 기존 서비스의 한계 | EarningWhisperer의 대안 |
| --- | --- | --- |
| 비용/접근성 | 고가의 구독료 (전문가 위주) | 개인 투자자도 접근 가능한 합리적 구조 |
| 시점 | 사후 리포트 중심 (이미 늦은 정보) | **실시간(Live)** 스트리밍 분석 및 대응 |
| 개인화 | 범용적인 정보 제공 | 사용자 포트폴리오 및 리스크 선호도 반영 |
| 분석 방식 | 블랙박스형 감성 지수 (단순 수치) | LLM 기반의 **명확한 판단 근거** 제시 |

---

### 2. 분석 데이터 및 기술 스택 (Technical Analysis)

#### 📥 입력 데이터 (Input)

- **데이터 소스:** 어닝콜 영상 및 오디오 스트리밍
- **전처리:** 영상/오디오 → 텍스트(STT) 변환
- **추가구현(선택):** 단순 텍스트 외에 **음성 성대의 변화량(톤, 떨림 등)**을 데이터화하여 비언어적 맥락 파악 시도

#### 🤖 모델 비교: LLM vs Traditional NLP

| 비교 항목 | LLM (Large Language Model) | NLP (Traditional / FinBERT) |
| --- | --- | --- |
| 장점 | 사전 학습 불필요, 고성능, **판단 근거 출력 가능** | 짧은 지연 시간(Latency), 낮은 운영 비용 |
| 단점 | 높은 비용, 지연 시간 발생, 결과의 일관성 부족 | 복잡한 라벨링 작업 필요, 데이터 의존도 높음 |

**선택 전략:** 하이브리드 — 로컬 FinBERT로 빠른 스캐닝 + 주요 구간 GPT API 심층 분석

---

### 3. 매매 로직 및 리스크 관리 (Trading Logic)

단순 시그널 발생 시 매수하는 것이 아니라, 사용자의 **재무 상태와 포트폴리오**를 고려한 *룰 엔진*을 거칩니다.

<aside>
💡

**규칙 기반 리스크 필터링 시스템 (Rule-Based Filter)**

1. **현금 비중 필터:** 가용 현금이 설정값 미만일 경우 주문 차단
2. **섹터 노출 필터:** 특정 섹터(예: IT) 비중이 높을 경우 매수 제한
3. **최대 베팅 필터:** 1회 주문당 최대 허용 금액 제한

**주문 흐름**

1. 어닝콜 시그널 수신
2. 사용자 계좌 상태 검증
3. 리스크 룰 엔진 통과
4. 최종 주문 실행
</aside>

---

### 4. 차별화 및 효용성 (Differentiation)

#### 💡 핵심 차별화 포인트

- **초개인화 라이브 알림:** 사용자의 관심 키워드 및 포트폴리오 연관 내용 실시간 푸시
- **흐름 시각화:** 타임라인 기반 감성 흐름도(Sentiment Flow) 및 스크립트 주요 구간 하이라이트
- **근거 중심:** “왜 이 주식이 유망한가?”에 대한 AI의 논리적 근거 요약 제공

#### 🌟 기대 효과

- 정보 비대칭성 해소 (기관 vs 개인)
- 심야 시간대 어닝콜 모니터링 피로도 감소

---

### 5. 서비스 아키텍처 (4-Layer Server)

1. **데이터 수집 서버:** 영상/음성 스트리밍 수집 및 STT 변환
2. **AI 추론 서버:** FinBERT(로컬) 또는 GPT API를 통한 감성 분석 및 요약
3. **비즈니스/매매 서버:** 리스크 필터링 로직 수행 및 증권사 API 연동
4. **프론트엔드 서버:** 리얼타임 대시보드 및 시각화 리포트 제공
