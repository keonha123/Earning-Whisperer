# 🖥️ Frontend (유저 대시보드) 요구사항 및 UI/UX 기획서

## 1. 모듈의 역할 및 목표
이 모듈은 EarningWhisperer의 유일한 고객 접점(Client-side)입니다. 
복잡한 AI 분석 결과와 퀀트 매매 내역을 일반 투자자가 직관적으로 이해하고 통제할 수 있도록, **모던 웹 대시보드**를 제공합니다. 핵심 목표는 '실시간 데이터의 지연 없는 시각화(WebSocket)'와 '사용자 자산 보호를 위한 직관적인 리스크 제어(UI/UX)'입니다. 법적 안전성을 위해 **모의투자 환경**으로만 구동됩니다.

## 2. 핵심 화면 및 기능 명세 (Core Pages)

### [Common UI] 전역 레이아웃 (Global Layout)
- **기능:** 모든 페이지에서 항상 노출되는 공통 네비게이션 및 알림 영역입니다.
- **구성 요소:**
  - **Top Header (상단 바):**
    - **전역 검색창:** 티커(Ticker)를 검색하여 관심종목으로 추가하거나 라이브 트레이딩 룸으로 즉시 이동합니다.
    - **글로벌 알림(🔔):** 관심종목의 어닝콜 시작, 백그라운드 자동매매 체결 시 실시간 알림(Toast/Badge)을 제공합니다.
  - **Left Sidebar (좌측 메뉴바):**
    - 대시보드 / 캘린더 / 매매 히스토리 / 설정 등 주요 페이지 네비게이션.

### [Page 1] 인증 (Auth)
- **기능:** 사용자 로그인 및 회원가입 화면.
- **특징:** 모든 API 요청 시 인증 토큰(JWT)을 헤더에 포함하여 통신합니다.

### [Page 2] 메인 대시보드 (Main Dashboard)
- **기능:** 로그인 직후 진입하는 포트폴리오 오버뷰.
- **구성 요소:**
  - **내 모의투자 요약:** 총 자산, 보유 현금, 실시간 수익률 및 자산 추이 그래프.
  - **보유 종목 리스트:** 현재 보유 중인 종목들의 평가 손익 현황.
  - **오늘의 일정:** 당일 어닝콜이 예정된 주요 타겟 기업 리스트.

### [Page 3] 관심종목 & 어닝콜 캘린더 (Watchlist)
- **기능:** 타겟 기업을 검색하고 관심종목으로 관리하는 화면.
- **구성 요소:**
  - **캘린더 뷰 (Table):** 관심종목의 현재가, 일일 등락률과 함께 **[다음 어닝콜 D-Day]**를 강조.
  - **Live 상태 표시:** 어닝콜이 시작된 종목은 최상단으로 이동하며 🔴 **LIVE** 배지가 점멸합니다.

### [Page 4] 🔴 라이브 트레이딩 룸 (Live Trading Room) - 🌟 핵심 화면
- **기능:** 특정 종목 클릭 시 전환되는 상세 화면. 백엔드로부터 WebSocket을 통해 실시간 데이터를 수신하여 뿌려줍니다.
- **구성 요소 (3분할 레이아웃):**
  - **[좌측] 차트 패널:** 해당 종목의 실시간 주가 틱 차트 및 호가창.
  - **[중앙] AI 실시간 분석 패널 (핵심):**
    - STT로 변환된 발언 텍스트가 타이핑 효과로 올라옴.
    - 상단에는 AI의 즉각적인 반응을 보여주는 **'Raw Score 텐션 미터기(게이지)'**가 빠르게 움직임.
    - 하단에는 백엔드가 계산한 노이즈가 제거된 **'EMA 추세선(Trend Line)'**이 꺾은선 그래프로 그려져 시스템의 신뢰성을 시각화.
  - **[우측] 시그널 피드 & 매매 패널:** LLM이 생성한 매매 근거(Rationale)가 피드에 쌓이며, 하단에 매수/매도 실행 버튼이 위치함.

### [Page 5] 📜 시그널 및 매매 히스토리 (Trade History)
- **기능:** AI 시그널 발생 내역과 실제 체결된 주문 내역을 대조하는 영수증 화면.
- **구성 요소:** 특정 시간에 AI가 **어떤 발언**을 근거로 **어떤 시그널**을 냈으며, 백엔드의 모의투자 엔진을 통해 **얼마에 체결**되었는지 투명하게 리스트업합니다.

### [Page 6] 설정 및 리스크 관리 (Settings & Docs)
- **기능:** 증권사 모의투자 API 연동 및 백엔드 매매 룰을 세팅하는 화면.
- **구성 요소:**
  - **증권사 연동:** 증권사 모의투자(Virtual) API 키 입력 및 암호화 저장. (실전투자 지원 안 함)
  - **🚨 리스크 관리 UI (백엔드와 동기화):**
    - 1회 매수 시 예수금 사용 비율 (%)
    - 특정 종목 최대 보유 비중 제한 (%)
    - 매수 쿨다운 타임 설정 (예: 5분 내 중복 시그널 무시)
    - *이 설정값들은 백엔드 룰 엔진으로 전송(`PUT`)되어 실제 자동매매 방어 로직에 적용됩니다.*

## 3. 핵심 UI/UX 정책: 트레이딩 모드 토글 (Trading Mode)
유저의 심리적 저항감을 낮추고 책임을 명확히 하기 위해 라이브 트레이딩 룸에 3단계 모드 스위치를 배치합니다.
1. **수동 모드 (Manual):** AI는 시그널 피드만 제공하며, 자동매매는 차단됩니다. 유저가 직접 버튼을 클릭하여 매매 API를 호출합니다.
2. **반자동 모드 (1-Click):** AI 매매 신호(EMA 통과) 발생 시 화면 중앙에 승인 팝업이 등장합니다. 유저가 [승인]을 누르거나 단축키(Enter)를 입력해야만 백엔드로 매매 요청이 전송됩니다.
3. **완전 자동 모드 (Auto-Pilot):** 유저가 '설정(Page 6)'에서 맞춘 리스크 관리 규정에 따라, 프론트엔드 개입 없이 백엔드가 백그라운드에서 알아서 모의 매매를 수행합니다.

## 4. 기술 스택 (Frontend)
- **Framework:** Next.js (React 18+)
- **Styling:** Tailwind CSS + Radix UI (또는 shadcn/ui)
- **State Management:** Zustand (유저 설정, 모드 토글 등 전역 상태 관리)
- **Real-time 통신:** `SockJS` + `@stomp/stompjs` (Spring Boot 백엔드의 WebSocket 채널 구독 및 실시간 데이터 수신)
