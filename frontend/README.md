# 🖥️ Frontend (유저 대시보드) 요구사항 및 UI/UX 기획서

## 1. 모듈의 역할 및 목표
이 모듈은 EarningWhisperer의 유일한 고객 접점(Client-side)입니다. 
복잡한 AI 분석 결과와 퀀트 매매 내역을 일반 투자자가 직관적으로 이해하고 통제할 수 있도록, **SaaS 형태의 모던 웹 대시보드**를 제공합니다. 핵심 목표는 '실시간 데이터의 지연 없는 시각화(WebSocket)'와 '사용자 자산 보호를 위한 직관적인 리스크 제어(UI/UX)'입니다.

## 2. 핵심 화면 및 기능 명세 (Core Pages)

### [Common UI] 전역 레이아웃 (Global Layout)
- **기능:** 모든 페이지에서 항상 노출되는 공통 네비게이션 및 알림 영역입니다.
- **구성 요소:**
  - **Top Header (상단 바):**
    - **전역 검색창:** S&P 500 종목명(Ticker)을 검색하여 즉시 관심종목으로 추가하거나 해당 종목의 라이브 트레이딩 룸으로 이동합니다.
    - **글로벌 알림(🔔):** 관심종목의 어닝콜 라이브가 시작되었거나, 백그라운드에서 자동매매가 체결되었을 때 실시간 알림(Toast 또는 Badge)을 띄워줍니다.
  - **Left Sidebar (좌측 메뉴바):**
    - 대시보드 / 관심종목 캘린더 / 매매 히스토리 / 설정 / 이용 가이드 등 주요 페이지로 이동하는 메인 네비게이션 역할을 합니다.

### [Page 1] 인증 (Auth)
- **기능:** 사용자 로그인 및 회원가입 화면.
- **특징:** 금융 서비스인 만큼, 로그인 후 세션이 안전하게 유지되어야 하며 모든 API 요청 시 토큰(JWT 등)을 포함해야 합니다.

### [Page 2] 메인 대시보드 (Main Dashboard)
- **기능:** 로그인 직후 가장 먼저 진입하는 오버뷰 화면.
- **구성 요소:**
  - **내 포트폴리오 요약:** 총 자산, 보유 현금, 실시간 수익률 및 자산 추이 그래프.
  - **보유 종목 리스트:** 현재 보유 중인 종목들의 평가 손익 현황.
  - **오늘의 일정:** 당일 어닝콜이 예정된 주요 기업 리스트업.

### [Page 3] 관심종목 & 어닝콜 캘린더 (Watchlist)
- **기능:** 타겟 기업을 검색하고 관심종목으로 관리하는 화면.
- **구성 요소:**
  - **캘린더 뷰 (Table):** 관심종목의 현재가, 일일 등락률과 함께 **[다음 어닝콜 D-Day]**를 강조해서 표시.
  - **Live 상태 표시:** 어닝콜이 시작된 종목은 리스트 최상단으로 이동하며 🔴 **LIVE** 배지가 점멸하여 진입을 유도합니다.

### [Page 4] 🔴 라이브 트레이딩 룸 (Live Trading Room) - 🌟 핵심 화면
- **기능:** 특정 종목 클릭 시 전환되는 상세 화면. 어닝콜 방송 중 실시간 데이터를 뿌려줍니다.
- **구성 요소 (3분할 레이아웃):**
  - **[좌측] 차트 패널:** 해당 종목의 실시간 주가 차트 및 호가창.
  - **[중앙] AI 실시간 분석 패널:** STT로 변환된 CEO의 발언이 타이핑 효과로 올라오며, 상단에 AI가 판별한 **'긍정/부정 텐션 미터기(Gauge Bar)'**가 실시간으로 움직임.
  - **[우측] 시그널 피드 & 매매 패널:** LLM이 생성한 매매 근거(Rationale)가 피드 형태로 쌓임. 하단에는 매수/매도 실행 버튼 위치.

### [Page 5] 📜 시그널 및 매매 히스토리 (Trade History)
- **기능:** AI 시그널 발생 내역과 실제 체결된 주문 내역을 대조하여 보여주는 영수증 화면.
- **구성 요소:**
  - 특정 날짜/시간에 AI가 **어떤 발언**을 근거로 **어떤 시그널**을 냈으며, 백엔드를 통해 **얼마에 체결**되었는지 투명하게 리스트업하여 시스템의 신뢰도를 제공합니다.

### [Page 6] 설정 및 리스크 관리 (Settings & Docs)
- **기능:** 증권사 API 연동 및 퀀트 봇의 안전장치를 세팅하는 화면.
- **구성 요소:**
  - **증권사 연동:** 모의투자/실전투자 API 키 입력 및 암호화 저장.
  - **설명서 (Docs):** 서비스 이용 가이드 링크.
  - **🚨 리스크 관리 UI (핵심):**
    - 1회 매수 시 예수금 사용 비율 (%)
    - 특정 종목 최대 보유 비중 제한 (%)
    - 매수 쿨다운 타임 설정 (예: 5분 내 중복 시그널 무시)

## 3. 핵심 UI/UX 정책: 트레이딩 모드 토글 (Trading Mode)
사용자의 심리적 저항감을 낮추기 위해, 라이브 트레이딩 룸(Page 4) 패널에 3단계 모드 스위치를 배치합니다.
1. **수동 모드 (Manual):** AI는 시그널 피드만 제공하며, 매매 버튼은 유저가 직접 클릭.
2. **반자동 모드 (1-Click):** AI 시그널 발생 시 중앙 팝업 등장. 유저가 [승인]을 누르거나 단축키를 입력해야 체결.
3. **완전 자동 모드 (Auto-Pilot):** 유저가 '설정'에서 맞춘 리스크 관리 규정에 따라 백그라운드에서 알아서 매매 수행.

## 4. 기술 스택 (Frontend)
- **Framework:** Next.js (React 기반)
- **Styling:** Tailwind CSS
- **State Management:** Zustand 또는 Redux (유저 포트폴리오 전역 상태 관리)
- **Real-time 통신:** WebSocket (또는 Server-Sent Events) - 백엔드에서 쏴주는 라이브 스크립트와 시그널 수신용.
