## 1. 모듈의 역할 및 목표
이 모듈은 EarningWhisperer 프로젝트의 '귀' 역할을 담당합니다. 
유튜브 라이브나 오디오 스트림에서 실시간으로 기업의 어닝콜(실적 발표) 음성을 수집하고, 이를 STT(Speech-to-Text) 모델을 통해 텍스트로 변환한 뒤, AI 추론 서버로 전달하는 것이 핵심 목표입니다.

## 2. 핵심 기능 요구사항 (Core Features)

### [Feature 1] 실시간 오디오 스트리밍 캡처
- 대상 기업의 어닝콜 라이브 방송(주로 YouTube Live 또는 웹 캐스트) 오디오를 실시간으로 가져와야 합니다.
- **제약사항:** 영상(Video)은 필요 없으며, 트래픽 최적화를 위해 오디오(Audio) 스트림만 추출해야 합니다.

### [Feature 2] STT (Speech-to-Text) 변환
- 수집된 오디오를 텍스트로 변환합니다.
- **추천 모델:** OpenAI의 `Whisper` (로컬 구동 또는 API 사용) 또는 빠르고 가벼운 파이썬 음성 인식 라이브러리.
- **제약사항:** 어닝콜은 주로 영어로 진행되므로 영문 인식률이 높아야 하며, 지연 시간(Latency)을 최소화해야 합니다.

### [Feature 3] 텍스트 청킹(Chunking) 및 데이터 전송
- 끊임없이 이어지는 STT 텍스트를 문맥이 끊기지 않는 선에서 **약 10~15초 단위의 조각(Chunk)**으로 나눕니다.
- 나누어진 텍스트를 조립하여 `AI Engine` 서버의 REST API로 전송(`POST`)합니다.

## 3. 입출력 명세 (I/O Specification)
- **Input:** YouTube Live URL 또는 실시간 오디오 스트림 소스
- **Output:** AI 엔진으로 보내는 HTTP POST Request (`docs/api-spec.md`의 파이프라인 1번 규격 엄수)

## 4. 기술 스택 (Python)
- `yt-dlp` 또는 `pafy`: 유튜브 라이브 오디오 스트림 추출용
- `ffmpeg`: 오디오 포맷 변환 및 처리용
- `openai-whisper` 또는 `SpeechRecognition`: STT 엔진
- `requests`: HTTP API 통신용
